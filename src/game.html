Terminals were really important before graphical user interfaces existed. They were the only way to comunicate with you computer. Everyone who wanted to use computers had to know how to use the command line. But things change.

Graphical user interfaces evolved. The first steps were hard but computer experts migrated to graphical user interfaces. For new users, everything became much simplier and the command line was almost forgotten. Almost. Why does it still exist? Even on Windows?

Because they are really powerfull. There are many reasons why this is so. The first one is that many great tools were made before graphical programms existed and are still used today. Programmers don't have a problem using the command line so they didn't bother to redesign everything to be graphical. Creating command line programs is much faster than designing graphical interfaces. If you need to calculate something, analyse some text or just test a new thesis, you will probably use the command line.

The terminal is simple to use. The correct syntax is always COMMAND [ARGUMENTS]. You type the command and press RETURN on your keyboard.

Every command has its help specification. You can see it by typing help COMMAND (example help pwd). The specification conists of: description, usage, example and Linux reference. The description tells you what is the command for, the usage and examples tell you how to use it and the Linux reference is the most similar command on the Linux terminal.

  • COMMAND: Command has no arguments. You just type the command in the terminal. Example: pwd
  • COMMAND ARGUMENT: Command needs an argument to work. Example: mkdir a_new_directory
  • COMMAND ARGUMENT1 ARGUMENT2: Command needs two arguments to work. Example: mv file to_new_location
  • COMMAND [ARGUMENT]: Command can have an optional argument but doesn't need one to operate. Example: ls or ls /home
  • COMMAND ARGUMENT1 [ARGUMENT2]: Command needs the first argument to wok but can also use an optional argument. The first argument allways needs to be specified. Also, the order is important. Example: edit file or edit protected_file password.
  • COMMAND [ARGUMENT1 [ARGUMENT2]]: Command doesn't need any arguments but you can specify two optional arguments. If you want to specify the second argument, you first need to specify the first.
