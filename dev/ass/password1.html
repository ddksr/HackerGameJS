How to crack passwords

Your assignment is to hack password protected files. You will have to try different aprochaes. You can learn more about them on the Learn more section. We have lodaded password protected files in your /tmp directory. In each task you will try to crack one password.

Find the directory in /tmp where the password protected files are stored. Go to that directory.
Try to open the passwords with the edit command. Don't leave your current directory. You will get a message that the file is not a text file.
Crack the fist file (file1) with simple guesing. When you know the content, send it to sensei. Use only numbers
Crack the second file (file2) with Brute force. Use brute command. When you know the file content, send it to sensei.
Crack the third file (file3) with a Dictionary attack. Use dict. When you know the file content, send it to sensei.

There are many ways to crack passwords.

  • Simple guesing
  • Brute force
  • Dictionary attack
  • Guesing using victims information
  • Searching for a user in known database dumps which were hacked by other hackers.

This method has a problem too. It is very slow. And it can possibly never crack the password. Imagine you would need to crack a password like 'abcdefgh' and you would only try lower letters. There are 26 lower letters in the English alphabet. You can generate 26 string of length 1, 26*26 strings of length 2, ... 26*26*26*26*26=265=11,881,376 strings of length 5, ... 268=208,827,064,576 strings of length 8 (like 'abcdefgh'). This is really a lot. And we used only lowercase letters. How about if we add upercase letters and numbers? We get 26+26+10=62 characters. 628 = 218,340,105,584,896. If the operation takes only a microsecond (which is 0.0000001 s) this would take 218,340,105,584,896 microseconds which is about 218,340,105 seconds = 3,639,001 minutes = 60,650 hours = 2,527 days = cca 7 years. Dou you have that much time?

Guessing using victims information

If you have any information about the victim it can really help you guess the correct password. It is very common that people use dates, their childrens or partners or pets names, favorite music artists etc. You can try victims date of birth, children names, wife or husband name, children names in combination, his first pet name. If a webpages enables hints it can be even easier to guess the hint and not the password.
