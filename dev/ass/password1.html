How to crack passwords

We have obtained three password protected files. They are all from three different persons who tried to lock a big secret into parts.

Your assignment is to hack password protected files. You will have to try different approaches. You can learn more about them on the Learn more section.

Find the directory in /tmp where the password protected files are stored. Go to that directory.
Try to open the passwords with the edit command. Don't leave your current directory. You will get a message that the file is not a text file.
Crack the fist file (amanda) with simple guessing. When you know the content, send it to sensei. Use only numbers
Crack the second file (silvia) with Brute force. Use brute command. When you know the file content, send it to sensei.
Crack the third file (franny) with a Dictionary attack. Use dict. When you know the file content, send it to sensei.

There are many ways to crack passwords.

  • Simple guessing
  • Brute force
  • Dictionary attack
  • Guessing using victims information
  • Searching for a user in known database dumps which were hacked by other hackers.

Simple guessing

Many people use simple passwords to protect their private content. Most common passwords are: password, 123456, 12345678, abc123, qwerty, monkey, letmein, dragon, 111111, baseball, iloveyou, trustno1, 1234567, sunshine, welcome, shadow, ashley, football, jesus ...

The thing is you just try to guess and sadly there is a large probability that you will succeeded. The problem with this method is that you cannot guess the password every time. Could you guess something like 100happyFaces?

Brute force

This method has a problem too. It is very slow. And it can possibly never crack the password. Imagine you would need to crack a password like 'abcdefgh' and you would only try lower letters. There are 26 lower letters in the English alphabet. You can generate 26 string of length 1, 26*26 strings of length 2, ... 26*26*26*26*26=265=11,881,376 strings of length 5, ... 268=208,827,064,576 strings of length 8 (like 'abcdefgh'). This is really a lot. And we used only lowercase letters. How about if we add uppercase letters and numbers? We get 26+26+10=62 characters. 628 = 218,340,105,584,896. If the operation takes only a microsecond (which is 0.0000001 s) this would take 218,340,105,584,896 microseconds which is about 218,340,105 seconds = 3,639,001 minutes = 60,650 hours = 2,527 days = cca 7 years. Do you have that much time?

Dictionary attack

This method is usually a lot more promising than Brute force. People usually don't use passwords like 'jzshjkahd' but something like 'apple' or 'lemonjuice'. Image you have a really really long list of words and you try each and every one of them if it is the correct password. Usually you also add common passwords on the top of the list.
