How to move around
This assignment is very important. You will learn:
  • how to move from one directory to another
  • how to list files in a directory
  • how to create a new direcotry

It is obligatory that you understand how to access a file on your computer. To access a file, you have to specify the path to the file. If you never used a terminal before and you don't understand paths, please check out Learn More. You will quickly see how easy it is! This is probably the longest tutorial. But I promise, others are shorter!

Type in pwd. You will see your current directory. Your current directory should be / which is the root of your file system.
You probably want to see which files and driectories are in your current directory. You have two options and the first one you already know. You can type tree in the terminal and see your whole file system. The second option is even simplier to read. Type ls and you will only see the files in your current working directory. To se the contents of a specific file you can submit an argument to the ls command. Type help ls to learn how to do that. Display the contents of the /bin directory.
Now change your working directory to /bin. Use cd. The schema for arguments is COMMAND ARGUMENT1 ARGUMENT2 ... Type cd bin or cd /bin
If you type ls you will see different files. That is because ls without arguments display the contents of the working directory. Change your working directory back to root. What is root? It is the parent directory from which every file is accessible. Use absolute path to change to your working directory. Type cd /
Now go back to bin using only relative paths. When you are finished, go back to root again but using relative path. Two dots are used to go back one directory Go back to root with cd ..
Now go to /tmp directory.
You can create directories yourself. The command is called mkdir (make directory). Create a directory named assignment. Go to your new directory. Use help mkdir for more information.
Great job. But the contents of /tmp get deleted everytime you power your computer. But you don't need the folder anyway. So creating a another directory wont mess up your machine. Create a directory in /tmp again from your current directory using relative paths. The name is not important. Remember the two dots? You can use them everywhere!

What is a directory?

Directory is a file that can contain other files and directories. They are called folders in Microsoft Windows operating systems.

What is a working directory?

A working directory is the directory you are currently in. To see your working directory, type pwd in the terminal.

Why are working directories important?

They are not really important. They are just here to help. Every file in a file system has an absolute path to it. If you are using Windows, the absolute path to Program Files folder is C:\Program Files.

If we know the absolute path, we know how to access it. We access it with C:\Program Files. Simple. But what about if the absolute path is very long? And we need to type it more than one time? Would you type C:\Folder 1\Folder 2\Folder 3\My Favorite File everytime you need your favorite file? I hope not!

Some smart (read lazzy) people found a solution how not to type everything everytime. If you change your working directory to C:\Folder 1\Folder 2\Folder 3, you can just access the file by typing My Favorite File.

OK. Let's stop with the Windows path schema and use Unix path. Why? It is not only supported on Unix-based operating systems but also on the world wide web (URLs). Why on the web? The wast myjority of web servers (computers that serve the web page) use Linux!

Let's examine Charlie's computer. Who is Charlie? Well, nobody really. Just our imagination. His current working directory is /home/charlie (the absolute path to the directory). Now he would like to switch the working directory to /home/simon/download. The command to do that (same on every operating system) is cd (change directory). He has two options:

  • Use absolute path: cd /home/charlie/download
  • Use relative path: cd download
Simple! And now he would like to go back. Again, he has two options:
  • Use absolute path: cd /home/charlie
  • Use relative path: cd .. (two dots mean bask, one dot means 'stay here')
And now he wishes to display file contents of file my_math_homework in directory /home/charlie/documents.
  • Use absolute path: cat /home/charlie/documents/my_math_homework
  • Use relative path: cat documents/my_math_homework

That's almost it. There are also many other possibilities. Imagine his working directory is /home and he would like to go to /bin. You already know how to do that with the absolute path. But how about with relative path? Simple: cd ../bin which means 'Go back one directory and then go into /bin'
