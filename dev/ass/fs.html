What is a directory?

Directory is a file that can contain other files and directories. They are called folders in Microsoft Windows operating systems.

What is a working directory?

A working directory is the directory you are currently in. To see your working directory, type pwd in the terminal.

Why are working directories important?

They are not really important. They are just here to help. Every file in a file system has an absolute path to it. If you are using Windows, the absolute path to Program Files folder is C:\Program Files. Windows systems separate files with '\' character and Unix-based systems with '/'. A web page URL is also an example of a absolute path. Example: http://www.google.si/maps.

If we know the absolute path, we know how to access it. We access it with C:\Program Files. Simple. But what about if the absolute path is very long? And we need to type it more than one time? Would you type C:\Folder 1\Folder 2\Folder 3\My Favorite File every time you need your favorite file? I hope not!

Some smart (read lazy) people found a solution how not to type everything every time. If you change your working directory to C:\Folder 1\Folder 2\Folder 3, you can just access the file by typing My Favorite File.

OK. Let's stop with the Windows path schema and use Unix path. Why? It is not only supported on Unix-based operating systems but also on the world wide web (URLs). Why on the web? The wast majority of web servers (computers that serve the web page) use Linux!

Let's examine Charlie's computer. Who is Charlie? Well, nobody really. Just our imagination. His current working directory is /home/charlie (the absolute path to the directory). Now he would like to switch the working directory to /home/charlie/download. The command to do that (same on every operating system) is cd (change directory). He has two options:

  • Use absolute path: cd /home/charlie/download
  • Use relative path: cd download
Simple! And now he would like to go back. Again, he has two options:
  • Use absolute path: cd /home/charlie
  • Use relative path: cd .. (two dots mean bask, one dot means 'stay here')
And now he wishes to display file contents of file my_math_homework in directory /home/charlie/documents.
  • Use absolute path: cat /home/charlie/documents/my_math_homework
  • Use relative path: cat documents/my_math_homework

That's almost it. There are also many other possibilities. Imagine his working directory is /home and he would like to go to /bin. You already know how to do that with the absolute path. But how about with relative path? Simple: cd ../bin which means 'Go back one directory and then go into /bin'

Copying and moving files

The copy command (cp) is simple. You just use specified the source (file or directory you wish to copy) as the first argument and the destination (the location and new or old name of the file or directory) as the second argument. This is the same for moving.

You can try this in every Terminal. If you use Windows, be careful to use dir instead of ls.
